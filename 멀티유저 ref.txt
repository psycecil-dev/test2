# 목적
- Supabase 기반 멀티유저/멀티세션 RAG 앱을 만든다.
- 기존 `multi-session-ref.py`를 멀티유저 버전으로 확장한다.

# 파일 구조(필수)
- 메인 앱: `multi-users-ref.py` (이미 있으면 지워줘. 그리고 .\prompt\10.multi-user\ 디렉토리에 만들어줘.)
- 의존성: `requirements.txt`

# 상세기능
- Supabase Auth 로그인 기능 추가 (login id/password 입력).
- Supabase 기반 세션 저장/로드/삭제/초기화 기능 추가.
- 사용자별 데이터 분리(RLS: Row Level Security 기반).
- 문서 임베딩/메시지/세션 저장 시 반드시 로그인 사용자 기준으로 처리.

# 사용자 회원가입 관리
- 필수 요구사항. sidebar에서 회원가입버튼을 누르면 별도의 페이지로 이동
- 회원가입 별도의 페이지:
  - email, pw 입력후, supabase가 바로 confirmation email을 보냄. 
  - 사용자가 cofirm을 하면 바로 email, pw 저장.
- 가입 완료 후 바로 앱 사용 가능해야 한다.

# 세션 관리 버튼
- `multi-session-ref.py`와 동일한 UX 유지:
  - 세션저장, 세션로드, 세션삭제, 화면초기화, 제목보정, vectordb
